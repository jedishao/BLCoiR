Bug 307858  [Markers] UIUpdateJob may throw SWTException if control has been disposedThe runInUIThread(IProgressMonitor) method looks something like this...

try {
  /* ... */
  if (viewer.getControl().isDisposed()) {
    return Status.CANCEL_STATUS;
  }
  /* ... */
} finally {
  viewer.getTree().setRedraw(true);
  /* ... */
}

The problem here is that the 'finally' block gets executed if the control is disposed. Since the control has been disposed, the setRedraw(boolean) invocation throws an SWTException.
