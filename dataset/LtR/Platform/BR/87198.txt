Bug 87198  Unable to create a MessageDialog instance outside of SWT threadVersion: 3.1.0
Build id: I20050301-1230

You used to be able to create an instance of a MessageDialog outside the UI
thead so long as open was called from inside the UI thread.  It looks like
change a change introduced in 1.28 (or thereabouts) of
org.eclipse.jface.window.Window has caused this to no longer work.  Was this
intentional?

I can easily update our code to create the dialog in the UI thread.
